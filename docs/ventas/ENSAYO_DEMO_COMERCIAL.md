# Ensayo de demo comercial

Objetivo: enseñar el resultado de LocalLift en diez minutos, validar encaje y
pedir una decision concreta sin recorrer el editor ni improvisar promesas.

Estado: guion y hoja de medicion preparados. Ensayo humano pendiente.

## Preparacion

- Abrir `index.html?presentation=true&view=mobile`.
- Tener disponible la oferta y el borrador de condiciones.
- Activar un cronometro visible.
- No mostrar el editor salvo que el prospecto lo pida.
- No afirmar que una integracion o publicacion existe si no esta configurada.

## Recorrido cronometrado

| Tiempo | Objetivo | Frase o accion obligatoria |
| --- | --- | --- |
| 0:00-1:00 | Contexto | Confirmar negocio, canal de captacion y principal problema al reservar. |
| 1:00-2:00 | Diagnostico | Resumir el problema observado en una frase y pedir confirmacion. |
| 2:00-5:00 | Resultado movil | Mostrar servicios, confianza, llamada, WhatsApp, mapa y solicitud de cita. |
| 5:00-7:00 | Operacion | Explicar como llegan leads y reservas y que revisa el negocio. |
| 7:00-9:00 | Oferta | Explicar alcance, plazo, precio, deposito y mantenimiento sin descuentos improvisados. |
| 9:00-10:00 | Cierre | Preguntar si tiene sentido reservar el hueco de fundador y acordar siguiente accion. |

Pregunta de cierre:

> Si te encaja este resultado y el alcance, ¿reservamos el hueco con el
> deposito y empezamos a recopilar el material?

## Criterios de aprobado

- [ ] Termina en 10:00 o menos.
- [ ] Incluye al menos tres preguntas de descubrimiento.
- [ ] El problema del prospecto se resume antes de enseñar la demo.
- [ ] La demo se enseña primero en movil.
- [ ] Se muestran llamada, WhatsApp, mapa y solicitud de cita.
- [ ] Se explica el flujo operativo sin prometer automatizaciones inexistentes.
- [ ] Se comunican precio, deposito, plazo y mantenimiento.
- [ ] Se formula una pregunta de cierre y una siguiente accion concreta.
- [ ] No se abre el editor ni se recorren funciones irrelevantes.
- [ ] No se usan datos ficticios como si pertenecieran al prospecto.
