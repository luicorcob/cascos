# Direccion de producto visual

## Objetivo

LocalLift Studio no debe venderse como un editor generico. Debe sentirse como una mesa de produccion que permite preparar, presentar y entregar una web comercial para un negocio local en una sola sesion.

La sensacion de "creado con IA" se consigue mediante velocidad, buenas decisiones predeterminadas y variaciones estructuradas. No depende de generar HTML arbitrario.

## Referencias estudiadas

- GrapesJS: https://github.com/GrapesJS/grapesjs
  - Referencia para lienzo visual, gestion de bloques, estilos y exportacion.
- Puck: https://github.com/puckeditor/puck
  - Referencia para edicion basada en componentes y datos estructurados.
- Craft.js: https://github.com/prevwong/craft.js
  - Referencia para construir editores extensibles sobre componentes controlados.
- Webstudio: https://github.com/webstudio-is/webstudio
  - Referencia para interfaz profesional de produccion web y control visual.
- Codrops: https://github.com/codrops
  - Referencia para composiciones experimentales, tipografia cinetica y layouts que no parecen plantillas.
- Atropos: https://github.com/nolimits4web/atropos
  - Libreria MIT incorporada para collages con profundidad y parallax tactil.
- Wix AI Website Builder: https://www.wix.com/ai-website-builder
  - Referencia para convertir una conversacion breve en una web editable.

## Decisiones aplicadas

- Barra global compacta con proyecto activo, herramientas operativas y entrega.
- Lienzo central con navegador simulado y cambio de viewport.
- Panel lateral organizado por intencion: dirigir, elegir sector, comprobar entrega y editar.
- Variaciones creativas curadas en lugar de cambios aleatorios sin control.
- Packs visuales con muestras de color para que cada direccion se entienda antes de aplicarla.
- Modo presentacion para enseñar la web al cliente sin que el editor distraiga.
- Interfaz neutral y profesional para que la identidad del negocio destaque.
- Motor de seis direcciones artisticas que cambia composicion, jerarquia, ritmo e imagen.
- Huella visual determinista por negocio para evitar resultados repetidos incluso usando `Auto unica`.
- Modo `Muy visual` que reduce texto visible y prioriza imagen, CTA y prueba social.

## Principios para futuras iteraciones

1. Cada nueva opcion debe reducir tiempo de entrega o aumentar la capacidad de cerrar una venta.
2. Los cambios visuales deben expresarse como decisiones comprensibles: premium, comercial, local, claro o impacto.
3. La preview debe seguir siendo el mismo resultado que se exporta.
4. La personalizacion debe conservar estructura, legibilidad, responsive y conversion.
5. Antes de añadir mas controles, conviene crear mejores presets y automatizaciones.

## Siguientes mejoras de mayor retorno

1. Captura guiada de brief que seleccione automaticamente sector, pack, CTA y secciones.
2. Galeria de 12 a 20 direcciones creativas curadas con miniaturas reales.
3. Generacion asistida de textos y reorganizacion de secciones usando datos estructurados.
4. Publicacion a dominio temporal con un clic para enviar la propuesta al cliente.
5. Comparador antes/despues y propuesta comercial exportable con precio, plazo y mantenimiento.
